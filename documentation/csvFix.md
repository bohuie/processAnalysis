# CSV File Fixer

## Overview

This script repairs and enhances CSV files generated by GitHub data extractors. It uses batch processing, parallel API calls, and intelligent caching to efficiently fix missing or incorrect data across multiple repositories.

## Features

- Batch processing of commits (50 at a time)
- Parallel API calls with configurable workers (default: 10)
- Intelligent caching system to minimize API requests
- Progress tracking with tqdm
- Automatic detection of CSV issues
- Preserves original files with "_fixed" suffix for new versions
- Handles multiple repositories in one execution
- Rate limit handling for GitHub API

## Functions Reference

### OptimizedCSVFixer Class

Main class that handles CSV fixing operations.

#### __init__(github_token, repo_owner, repo_name)

Initialize the CSV fixer with optional GitHub authentication.

**Parameters:**
- `github_token` (str, optional): GitHub API token
- `repo_owner` (str, optional): Repository owner username/organization
- `repo_name` (str, optional): Repository name

#### fix_commits_csv_optimized(input_file, output_file)

Fix commit CSV files with batch processing and caching.

**Parameters:**
- `input_file` (str): Path to input CSV file
- `output_file` (str, optional): Path to output file (auto-generated if not provided)

**Returns:**
- str: Path to the fixed output file

**Fixes Applied:**
- Missing commit dates
- Missing or incorrect author names
- Missing file paths
- Missing line addition/deletion counts

#### fix_pr_csv_optimized(input_file, output_file)

Fix pull request CSV files with batch reviewer fetching.

**Parameters:**
- `input_file` (str): Path to input CSV file
- `output_file` (str, optional): Path to output file (auto-generated if not provided)

**Returns:**
- str: Path to the fixed output file

**Fixes Applied:**
- Missing `is_up_to_date` field
- Missing `has_conflicts` field
- Missing or incomplete `reviewers` list
- Missing `num_reviewers` count

#### scan_and_fix_all_repos(base_directory)

Automatically scan and fix all repository CSVs in a directory.

**Parameters:**
- `base_directory` (str): Path to directory containing repository folders

**Returns:**
- dict: Summary with keys:
  - `repos_scanned`: Number of repositories processed
  - `repos_fixed`: Number of repositories with fixes applied
  - `total_files_fixed`: Total CSV files fixed
  - `repo_details`: List of per-repository results
  - `start_time`: Execution start timestamp

### Helper Functions

#### _make_request(url)

Make GitHub API request with error handling and rate limit management.

**Parameters:**
- `url` (str): GitHub API endpoint URL

**Returns:**
- requests.Response or None: API response if successful

#### _fetch_commit_details_batch(commit_shas)

Fetch details for multiple commits in parallel with caching.

**Parameters:**
- `commit_shas` (list): List of commit SHA hashes

**Returns:**
- dict: Mapping of SHA to commit data

#### _fetch_reviewers_batch(pr_ids)

Fetch reviewers for multiple pull requests in parallel.

**Parameters:**
- `pr_ids` (list): List of pull request IDs

**Returns:**
- dict: Mapping of PR ID to list of reviewer usernames

#### _check_csvs_need_fixing(repo_dir)

Quick analysis to determine if CSVs need fixing.

**Parameters:**
- `repo_dir` (Path): Path to repository directory

**Returns:**
- dict: Analysis results with keys:
  - `needs_fix`: Boolean indicating if fixes are needed
  - `files_to_fix`: List of filenames requiring fixes
  - `issues`: List of specific issues found

## Expected CSV Structures

### Commits CSV

Required columns:
- `commit_sha`: Unique commit identifier
- `commit_date`: ISO format timestamp
- `author`: Commit author username
- `file_path`: Path to modified file
- `lines_added`: Number of lines added
- `lines_deleted`: Number of lines deleted

### Pull Requests CSV

Required columns:
- `pr_id`: Pull request number
- `description`: PR description text
- `was_behind_at_merge`: Boolean or numeric value
- `was_up_to_date_at_merge`: Boolean value
- `mergeable_state`: GitHub mergeable state

Added columns:
- `is_up_to_date`: Derived from merge status
- `has_conflicts`: Derived from mergeable state
- `reviewers`: Comma-separated list of reviewer usernames
- `num_reviewers`: Count of reviewers

## Performance Features

### Caching System

- Commit data cached by SHA hash
- PR reviewer data cached by PR ID
- Cache expires after 1 hour (configurable)
- Cache statistics reported: hits vs misses

### Batch Processing

- Commits processed in batches of 50
- Reduces API overhead
- Improves processing speed for large datasets

### Parallel Execution

- Up to 10 concurrent API requests
- Thread pool executor for safe parallelism
- Configurable worker count

## Error Handling

The script includes comprehensive error handling:

- Gracefully handles missing files
- Continues processing after individual errors
- Reports errors per repository without stopping
- Timeout protection for API calls (30 seconds)
- Rate limit detection and automatic retry

## Conflict Resolution Logic

### is_up_to_date Field

Derived from:
1. `was_behind_at_merge` field (if value is 0, then True)
2. Falls back to `was_up_to_date_at_merge` field

### has_conflicts Field

Derived from `mergeable_state`:
- `dirty`, `conflicting`, `unstable` results in True
- `clean`, `mergeable` results in False

### Reviewers List

Compiled from:
1. Pull request review comments
2. Issue comments on the PR
3. PR author excluded from reviewer list
4. Results sorted alphabetically

## Notes

- Original CSV files are never modified
- Fixed files use "_fixed" suffix
- Skips files already containing "_fixed" in filename
- Automatically searches common directory locations if path not specified
- GitHub authentication highly recommended for large datasets
- Cache is cleared between repositories to prevent memory issues
- Empty or malformed rows are preserved but not fixed

## Troubleshooting

### No data directory found

**Error:** `Could not find data/csv directory!`

**Solution:** Specify the directory path explicitly:
```bash
python csvFix.py /full/path/to/data/csv
```

### Rate limit exceeded

**Error:** HTTP 403 responses from GitHub API

**Solution:** 
- Add GitHub token to `.env` file
- Reduce `MAX_WORKERS` value
- The script automatically pauses briefly on rate limits

### Slow processing

**Performance issue:** Script takes too long

**Solution:**
- Ensure GitHub token is configured for higher rate limits
- Increase `BATCH_SIZE` for fewer API calls
- Increase `MAX_WORKERS` for more parallelism (be cautious of rate limits)

### Missing dependencies

**Error:** `ModuleNotFoundError`

**Solution:**
```bash
pip install requests python-dateutil python-dotenv tqdm
```

## Customization

### Adjust Batch Size
```python
fixer = OptimizedCSVFixer()
fixer.BATCH_SIZE = 100  # Process 100 commits at once
```

### Adjust Worker Count
```python
fixer = OptimizedCSVFixer()
fixer.MAX_WORKERS = 5  # Use 5 parallel workers
```

### Add Custom Fixes

Extend the `fix_commits_csv_optimized` or `fix_pr_csv_optimized` methods to add custom data transformations.

## API Rate Limits

### Without Authentication
- 60 requests per hour
- Suitable for small datasets only

### With Authentication
- 5,000 requests per hour
- Recommended for production use
- Required for large repositories
